## 11. ISSUES FOUND
### Code Quality Issues
- **DRY Violations**: Component injection via JS for Header/Footer bypasses static generation, which is bad for SEO and page load speed. Minor elements like `fab-container` are copy-pasted everywhere instead of using templates.
- **Tailwind via CDN**: The CDN script (`cdn.tailwindcss.com`) is injected on every page. This is meant for development only and sends a huge JS payload to the client. It needs to be compiled to a static CSS file.
- **Inline Styles**: Pages have extensive inline styles (e.g. `style="background: linear..."`) which makes overriding and theming difficult.
### Asset Issues
- **Image Optimization**: Images are mostly heavy `.jpg` or `.png` formats. Many are several MBs in size. They do not use `loading="lazy"` or modern formats like `.webp`/`.avif`.
- **Missing Dimensions**: `<img>` tags lack explicit `width` and `height` attributes, leading to Layout Shifts (CLS).
### SEO Issues
- **JS-Rendered Navigation**: Search engine crawlers without JS capabilities won't see the site's main navigation links or footer content.
- **Missing Meta Tags**: Pages lack Open Graph (`og:title`, `og:image`), Twitter Cards, and Canonical tags.
### Accessibility Issues
- Many buttons, icons, and SVG elements lack `aria-label` attributes.
- Form inputs occasionally lack linked `<label>` elements.
- Potential contrast issues on text overlaid onto hero background images.

## 12. MIGRATION READINESS NOTES
- **Framework Recommendation**: This site is highly suitable for a Static Site Generator (SSG) like **Astro**, **Next.js**, or **Nuxt**. Moving to Astro would allow keeping the exact same HTML/Tailwind classes while converting the JS Header/Footer into proper server-rendered components.
- **CSS Migration**: The custom CSS in `css/style.css` and `css/about_custom.css` should be ported into Tailwind config (`tailwind.config.js`) or `@apply` directives to unify the styling system.
- **Asset Pipeline**: Image optimization needs to be automated during build time (e.g., using `astro:assets` or `next/image`) to resize, compress, and serve WebP images automatically.
- **Routing**: Moving to a framework will clean up URLs from `/about.html` to just `/about`.
